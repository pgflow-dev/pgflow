## Core Concepts

pgflow combines three key components:

- **SQL Core Engine** - Data model and state machine in your Postgres database
- **TypeScript DSL** - Developer-friendly API for defining workflows with type safety
- **Edge Worker** - Task executor that runs on Supabase Edge Functions

## How does it work?

pgflow represents workflows as Directed Acyclic Graphs (DAGs):

1. Define steps connected by dependencies in TypeScript
2. Steps without dependencies start automatically
3. Steps with dependencies start once all dependencies complete
4. Edge Worker executes the steps by polling for ready tasks
